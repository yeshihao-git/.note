**ai**: 要让“生命周期”这个概念通俗易懂，最好的方法是**用类比**。

你可以把一个 Vue 组件想象成一个**“演员”**，他从被导演选中到最后谢幕离开舞台，经历了一系列固定的步骤。

建议你在 [[vue.js]] 笔记的 `## 生命周期` 下按照以下结构编写：

---

### 建议编写内容：

## 生命周期
**通俗理解**：组件就像一个“演员”，从出生（创建） $\rightarrow$ 上台（挂载） $\rightarrow$ 表演（更新） $\rightarrow$ 下台（销毁），在每个阶段，Vue 都会提供一个“钩子函数”，让你在特定时间点执行特定的代码。

### 1. 核心阶段分解

- **创建阶段 (Setup)** $\rightarrow$ **【演员在后台准备】**
	- 在 `<script setup>` 中，代码本身就是创建阶段。
	- **做什么**：初始化数据、定义变量。此时组件还没在页面上显示。
- **挂载阶段 (onMounted)** $\rightarrow$ **【演员正式登台】**
	- 组件已经渲染到页面上，用户能看到了。
	- **做什么**：发送网络请求获取数据、操作 DOM 元素、启动定时器。
- **更新阶段 (onUpdated)** $\rightarrow$ **【演员在台上换装/改词】**
	- 响应式数据发生了变化，页面重新渲染了。
	- **做什么**：在数据更新后执行某些操作（不常用，慎用，避免死循环）。
- **销毁阶段 (onUnmounted)** $\rightarrow$ **【演员谢幕下台】**
	- 组件被移除，不再显示。
	- **做什么**：清理工作。比如清除定时器 `clearInterval`、取消事件监听，防止内存泄漏。

### 2. 快速对照表

| 钩子函数 | 对应阶段 | 通俗含义 | 常用场景 |
| :--- | :--- | :--- | :--- |
| `setup` | 创建 | 准备中 | 初始化变量 |
| `onMounted` | 挂载 | 已上台 | **API 请求、DOM 操作** |
| `onUpdated` | 更新 | 变样子 | 响应数据变化后的处理 |
| `onUnmounted` | 销毁 | 已下台 | **清除定时器、解绑事件** |

### 3. 代码示例
```js
<script setup>
import { onMounted, onUnmounted, ref } from 'vue'

const timer = ref(null)

// 1. 挂载：组件刚上台
onMounted(() => {
  console.log('组件已挂载，开始请求数据...')
  timer.value = setInterval(() => {
    console.log('定时器运行中...')
  }, 1000)
})

// 2. 销毁：组件准备下台
onUnmounted(() => {
  console.log('组件已销毁，清理现场...')
  clearInterval(timer.value) // 必须清理，否则会一直运行导致卡顿
})
</script>
```

---

### 💡 为什么这样写更好？
1. **类比法**：用“演员”类比，把抽象的 `Mount/Unmount` 变成了具体的 `上台/下台`，降低了理解门槛。
2. **场景化**：不仅告诉用户是什么，还告诉用户**“做什么”**（比如 `onMounted` 适合发请求，`onUnmounted` 适合清定时器），这才是开发中最需要的知识。
3. **结构化**：从“概念 $\rightarrow$ 阶段 $\rightarrow$ 表格 $\rightarrow$ 代码”，符合人类从浅入深的认知逻辑。
