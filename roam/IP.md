# 地址划分
## 传统ABC类地址

**what：**
将 IP 地址分为 网络号+主机号
- A类 前8位为网络号， 其中前缀1位值为0  ，用于区分网络类型
  0  （1位） 网络号（ 7位） 主机号（24位）

- B类 前16位为网络号，其中前缀2位值为10 ，用于区分网络类型
  10 （2位） 网络号（14位） 主机号（16位）

- C类 前24位为网络号，其中前缀3位值为110，用于区分网络类型
  110（3位） 网络号（21位） 主机号（ 8位）

- D类 前缀4位值为1110，其他未定义

![[Pasted image 20260519182429.png|337]]

**划分子网方式** ：取部分主机号作为子网号。具体做法：采用 子网掩码 区分出子网号

```
B类网络 ：172.16.0.0，前16位为网络号
子网掩码：255.255.192.0，这里划分出的网络号为18，因此后面的2位为子网号
```

## 无分类地址CIDR

**what：**
a.b.c.d/x 前x位为网络号，剩下的为主机号

**why：** 
解决传统ABC类地址数量不够的问题

**划分子网方式** ：天生具备

### 子网

**what：**
是局域网的内部划分

```
某公司局域网 的 IP池 为 192.168.0.0/16
子网A：192.168.1.0/24
子网B：192.168.2.0/24
```

# IP分片与重组

**what：**
大 IP 数据报超链路 MTU 时，拆分成带标志位和偏移量的小分片发送，接收端按偏移量重组为完整数据报

> **标志位的作用**：
> 1. 表示该分片是否为最后一片
> 2. 表示该数据报是否允许分片

> **偏移量的作用**：
> 该分片中的数据部分在原数据报数据部分中的起始位置

**why**：
适配不同网络 MTU，确保数据能传输，同时通过重组还原原始数据供上层使用

**what**： MTU
MTU 是最大传输单元，指网络链路能传输的最大数据帧大小。常见网络的 MTU 大小：
1. 以太网：1500 字节
2. PPP 链路：1492 字节
3. ATM 网络：53 字节
4. 光纤网络：1500 字节及以上

**why**：MTU
限制单次传输数据量，平衡传输效率与可靠性

> - 假设不限制 MTU，则传输一次数据时间很长，失败后代价很大；
> - MTU 设置太小，会导致一个完整数据被拆成超多小分片，每个分片都要加 IP 首部，增加了额外的头部开销
