# 代码规范
## 命名规范
### 变量

1. 首字母 小写，驼峰
```cpp
int value; 
int myVariableName;     // 驼峰
int my_variable_name;   // 蛇形（也可以）
```
2. **全局变量** ： `g_` 前缀
3. **静态局部变量** ： `s_` 前缀

### 函数

1. 首字母 小写，驼峰
```cpp
int function();
int myFunctionName();   // 驼峰
int my_function_name(); // 蛇形
```

### 类、结构体

1. 首字母 大写，驼峰
```cpp
Customer
```
2. **类私有成员变量** ： `m_` 前缀
3. **类 `set` 函数** ： `set` 前缀
4. **类 `get` 函数** ： `无` 或 `get` 前缀

#### 接口

1. 首字母 `I` ，驼峰
```cpp
IShapeFactory // 接口
ShapeFactory  // 实现
```

### 命名空间

1. 首字母 大写，驼峰
2. **层级命名**：`公司/组织::项目/库::模块`（可以选择去掉 `公司/组织` 或 `模块`）
```cpp
namespace Tencent :: GameEngine :: Render { ... } // 完整三层
namespace GameEngine :: Render { ... }            // 省略组织层
namespace Tencent :: GameEngine { ... }           // 省略模块层
```

### 枚举

1. **枚举类型**：首字母 大写，驼峰
2. **枚举成员**：`全大写 + 下划线`
```cpp
// C 风格枚举（无作用域 enum）
enum Color {
  COLOR_RED,
  COLOR_GREEN,
  COLOR_BLUE
};

// C++ 风格枚举
enum class Color {
  Red,
  Green,
  Blue
};
```

### 类型别名

1. 首字母 大写，驼峰
```cpp
// using 别名（新）
using UserId = uint64_t;
using BufferView = std::span<char>;

// typedef 别名（旧）
typedef std::vector<int> IntVector;
```

> **番外**：C 语言写法
> `_t`、`_type` 结尾（eg：`size_t`）

### 模板
#### 类型模板参数

1. 首字母 大写，简单名称常用 `T`、`U`、`V`
2. 语义化名称用 驼峰
```cpp
// 简单名称
template<typename T, typename U>
void Swap(T& a, U& b);

// 语义化名称
template<typename ElementType>
class Vector;
```

#### 非类型模板参数

1. 首字母 大写，简单名称常用 `N`、`M`
2. 语义化名称用 驼峰
```cpp
// 简单名称
template<int N, int M>
class Matrix;

// 语义化名称
template<std::size_t BufferSize>
struct RingBuffer;
```

## 避免 ”魔术数“ 

```java
public List<int[]> getThem() {
    List<int[]> list1 = new ArrayList<int[]>();
    for (int[] x : theList)
        if (x[0] == 4) // 这里的 4 的语义让人摸不着头脑
            list1.add(x);
    return list1;
}
```

重命名变量、函数名，并使用[[枚举#枚举|枚举]]修改后，语义清晰了很多

```java
public List<int[]> getFlaggedCells() {
    List<int[]> flaggedCells = new ArrayList<int[]>();
    for (int[] cell : gameBoard)
        if (cell[STATUS_VALUE] == FLAGGED) // 此时我们知道 4 表示 “已标记”
            flaggedCells.add(cell);
    return flaggedCells;
}
```

更好的做法是使用 函数，代码更简洁，语义更清晰

```java
public List<Cell> getFlaggedCells() {
    List<Cell> flaggedCells = new ArrayList<Cell>();
    for (Cell cell : gameBoard)
        if (cell.isFlagged()) // 这里使用 isFlagged() 函数
            flaggedCells.add(cell);
    return flaggedCells;
}
```

## 函数应该短小

1. `if`、`else`、`while`、`switch` 语句等，其中的代码块应只有一行
2. 块内行 应是一个函数调用语句，且函数名能表达一定的语义

重构前：
```java
public static String renderPageWithSetupsAndTeardowns(
    PageData pageData, boolean isSuite
) throws Exception {
    boolean isTestPage = pageData.hasAttribute("Test");
    if (isTestPage) {
        WikiPage testPage = pageData.getWikiPage();
        StringBuffer newPageContent = new StringBuffer();
        includeSetupPages(testPage, newPageContent, isSuite);
        newPageContent.append(pageData.getContent());
        includeTeardownPages(testPage, newPageContent, isSuite);
        pageData.setContent(newPageContent.toString());
    }
    return pageData.getHtml();
}
```
重构后：
```java
public static String renderPageWithSetupsAndTeardowns(
    PageData pageData, boolean isSuite) throws Exception {
    if (isTestPage(pageData))
        includeSetupAndTeardownPages(pageData, isSuite); // 块内行为一个函数调用，且具有语义
    return pageData.getHtml();
}
```

## 函数应该只做一件事

要确保函数只做一件事，即需保证 ==函数中的语句应该在同一抽象层级上==

> 例如 一个函数中有 `getHtml()` 位于较高抽象层的概念，又有 `String pagePathName = PathParser.render(pagePath)` 位于中间抽象层的概念，还有 `.append("\n")` 位于相当低的抽象层的概念

如何判断函数中语句的层级？==使用 TO 原则，自顶向下阅读==
程序就像是一系列 TO 起头的段落，每一段都描述当前抽象层级，并引用位于下一抽象层级的后续 TO 起头段落

> To include the setups and teardowns, we include setups, then we include the test page content, and then we include the teardowns.（要容纳设置和分拆步骤，就先容纳设置步骤，然后纳入测试页面内容，再纳入分拆步骤）
> 
> To include the setups, we include the suite setup if this is a suite, then we include the regular setup.（要容纳设置步骤，如果是套件，就纳入套件设置步骤，然后再纳入普通设置步骤）
> 
> To include the suite setup, we search the parent hierarchy for the "SuiteSetUp" page and add an include statement with the path of that page.（要容纳套件设置步骤，先搜索 "SuiteSetUp" 页面的上级继承关系，再添加一个包括该页面路径的语句）

## 函数参数应该尽量少

==函数参数数量最佳为 0，其次 1，再次 2==，避免 3 及以上
原因在于 参数越多，函数的语义越难以理解（在没有注释的情况下）

> `writeField(name)` 比 `writeField(ouputStream, name)` 更好懂

函数参数数量为 2、3 或以上，则可以==将一些参数封装为类==，以减少参数数量
```cpp
Circle makeCircle(double x, double y, double radius);
Circle makeCircle(Point center, double radius); // 此处将 x、y 封装为类 Point
```

## 使用异常替代返回错误码

没有使用 [[异常处理#异常处理|异常处理]] 的时候，==代码逻辑和错误处理杂糅在一起==了
```cpp
if (deletePage(page) == E_OK) {
    if (registry.deleteReference(page.name) == E_OK) {
        if (configKeys.deleteKey(page.name.makeKey()) == E_OK) {
            logger.log("page deleted");
        } else {
            logger.log("configKey not deleted");
        }
    } else {
        logger.log("deleteReference from registry failed");
    }
} else {
    logger.log("delete failed");
    return E_ERROR;
}
```

使用异常处理后，代码逻辑与错误处理解耦
```cpp
try { // 代码逻辑
    deletePage(page);
    registry.deleteReference(page.name);
    configKeys.deleteKey(page.name.makeKey());
}
catch (Exception e) { // 错误处理
    logger.log(e.getMessage());
}
```

根据 [[代码规范#函数应该短小|函数应该短小]] 原则，再次优化；同时错误处理函数应该只做一件事不做其他事 `public void delete(Page page)`，即错误处理
```cpp
public void delete(Page page) { // 错误处理函数只做一件事，即错误处理
    try {
        deletePageAndAllReferences(page); // 函数应该短小原则
    }
    catch (Exception e) {
        logError(e); // 函数应该短小原则
    }
}

private void deletePageAndAllReferences(Page page) throws Exception {
    deletePage(page);
    registry.deleteReference(page.name);
    configKeys.deleteKey(page.name.makeKey());
}

private void logError(Exception e) {
    logger.log(e.getMessage());
}
```

## 过程式代码 和 面向对象代码

**过程式代码**
1. 难以添加新类，因为必须修改所有函数
2. 适合：需要添加新数据类型而不是新函数
```java
// 数据类型
public class Square {
    public Point topLeft;
    public double side;
}

public class Rectangle {
    public Point topLeft;
    public double height;
    public double width;
}

public class Circle {
    public Point center;
    public double radius;
}

// 操作 数据类型的类
public class Geometry {
    public final double PI = 3.141592653589793;
	
	// 函数
    public double area(Object shape) throws NoSuchShapeException
    {
        if (shape instanceof Square) {
            Square s = (Square) shape;
            return s.side * s.side;
        }
        else if (shape instanceof Rectangle) {
            Rectangle r = (Rectangle) shape;
            return r.height * r.width;
        }
        else if (shape instanceof Circle) {
            Circle c = (Circle) shape;
            return PI * c.radius * c.radius;
        }
        throw new NoSuchShapeException();
    }
}
```

**面向对象代码** 
1. 难以添加新函数，因为必须修改所有类
2. 适合：需要添加新函数而不是新数据类型
```java
// 数据类型
public class Square implements Shape {
    private Point topLeft;
    private double side;
	
	// 函数
    public double area() {
        return side*side;
    }
}

public class Rectangle implements Shape {
    private Point topLeft;
    private double height;
    private double width;

    public double area() {
        return height * width;
    }
}

public class Circle implements Shape {
    private Point center;
    private double radius;
    public final double PI = 3.141592653589793;

    public double area() {
        return PI * radius * radius;
    }
}
```

## 测试方法：构造-操作-检验 模式

每个测试函数分为三步：
1. **构造**：构造测试数据
2. **操作**：操作测试数据
3. **检验**：检查操作结果是否符合期望
```java
public void testGetDataAsXml() throws Exception {
	// 构造
    makePageWithContent("TestPageOne", "test page");
	
	// 操作
    submitRequest("TestPageOne", "type:data");
	
	// 检验
    assertResponseIsXML();
    assertResponseContains("test page", "<Test");
}
```
