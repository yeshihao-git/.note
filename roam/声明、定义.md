# 声明

**what**：
告诉编译器某个标识符（符号）存在
```cpp
int add(int x, int y); // 声明 add
```

## 前向声明

**what**：
（一种 不完全类型）告诉编译器某个 标识符的存在 ，后续再提供标识符的定义 
```cpp
int add(int x, int y); // 前向声明

int main() {
    std::cout << "The sum of 3 and 4 is: " << add(3, 4) << '\n';  // this works because we forward declared add() above
    return 0;
}

int add(int x, int y) { // even though the body of add() isn't defined until here
    return x + y;
}
```
1. 解决 循环依赖 问题，见 std::shared_ptr
2. 用于 *多文件程序链接* ，更优解：使用 头文件

```cpp
// main.cpp文件
int add(int x, int y); // needed so main.cpp knows that add() is a function defined elsewhere

int main()
{
    std::cout << "The sum of 3 and 4 is: " << add(3, 4) << '\n';
}
```

```cpp
// add.cpp文件
int add(int x, int y)
{
    return x + y;
}
```

## 不完全类型

**what**：
声明但未定义的类型

有哪些 ？
1. 前向声明 类类型（结构体、类、联合体）、变量
2. 指针 和 引用

# 定义

**what**：
是声明，且有对应实现
```cpp
int add(int x, int y) { // 定义 add
    int z{ x + y };
    return z;
}

int x;                 // 声明+定义 x
```

## 单一定义规则

**what**：
一个对象或函数在一个文件或程序内不能超过一个定义。具体来说：
1. 所有翻译单元 中 函数、变量 定义只能有 一个
2. 多个翻译单元 中 类类型（结构体、类、联合体）、模板 、inline函数、inline变量 定义可以有 多个 （函数模板实例化的函数是隐式inline的）
