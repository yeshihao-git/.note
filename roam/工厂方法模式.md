# 工厂方法模式

**what：**
一种创建型设计模式， 将对象的创建和使用分离，在基类中提供一个创建对象的方法， 派生类决定实例化对象的类型
1. 一个工厂生产一种产品
2. 降低系统耦合度（添加新类型只需修改工厂类），符合开闭原则
3. 易于维护（创建逻辑改变不会影响客户端代码）
4. 适合 ==产品需要扩展== 的情况

**how**：
1. 抽象工厂依赖抽象产品
2. 一个抽象产品派生多个具体产品
3. 一个抽象工厂派生多个具体工厂
4. 每个具体工厂生产一个具体产品

```cpp
#include <iostream>
#include <string>
#include <memory>

// Product
class Product
{
public:
    virtual ~Product() = default;
    virtual std::string someOperation() = 0;
};

// ConcreteProductA
class ConcreteProductA : public Product
{
public:
    std::string someOperation() override
    {
        return "concreteProductA";
    }
};

// ConcreteProductB
class ConcreteProductB : public Product
{
public:
    std::string someOperation() override
    {
        return "concreteProductB";
    }
};

// Creator
class Factory
{
public:
    virtual ~Factory() = default;
    virtual std::unique_ptr<Product> createProduct() = 0;

    // 不关注创建，直接使用产品
    void useProductToDoSomething()
    {
        std::unique_ptr<Product> product = this->createProduct();
        std::cout << "now, using " << product->someOperation() << std::endl;
    }
};

// ConcreteCreatorA
class ConcreteFactoryA : public Factory
{
public:
    std::unique_ptr<Product> createProduct() override
    {
        return std::make_unique<ConcreteProductA>();
    }
};

// ConcreteCreatorB
class ConcreteFactoryB : public Factory
{
public:
    std::unique_ptr<Product> createProduct() override
    {
        return std::make_unique<ConcreteProductB>();
    }
};

void clientCode(std::unique_ptr<Factory> factory)
{
    // 使用工厂方法创建后使用产品
    std::unique_ptr<Product> product = factory->createProduct();
    std::cout << product->someOperation() << std::endl;

    // 不关注创建，直接使用产品
    factory->useProductToDoSomething();
}

int main(int, char **)
{
    std::unique_ptr<Factory> factoryA = std::make_unique<ConcreteFactoryA>();
    clientCode(std::move(factoryA));

    std::unique_ptr<Factory> factoryB = std::make_unique<ConcreteFactoryB>();
    clientCode(std::move(factoryB));
}
```
