# 常量

**what**：
值不可修改 的实体量，包含：
1. 字面量（无标识符的直接常量值）
2. 命名常量（带标识符的 const/constexpr 变量）
	- 运行时常量：const常量
    - 编译时常量：constexpr常量
```cpp
// 命名常量
const int MAX = 100;
constexpr double PI = 3.14;

// 字面量
100, 3.14, "hello", true;
```

## 常量表达式

**what**：
编译期可计算的表达式。常见的常量表达式：
1. 字面量
2. 常量表达式 和 运算符 组合成的常量表达式（eg：3 + 4）
3. 用常量表达式初始化的 const int 
```cpp
const int a {10};       // 由于历史原因，NOTE 它是常量表达式，能够在编译时计算
int c {9};              // 非 常量表达式
const int d {c};        // 非 常量表达式
const double b {10.0};  // 非 常量表达式
```
4. constexpr变量
5. constexpr函数
6. constexpr lambdas
7. 非类型模板参数
8. 枚举成员
9. 类型特性 `<type_traits>`

## 字面量

**what**：
直接嵌入到源代码中的 constexpr 值，是 编译时常量，在 编译期初始化 ，具有静态存储期
1. 字面量有 默认类型 ，可根据其 value 推导出的 type

|字面值|示例|默认字面值类型|备注|
|---|---|---|---|
|整型值|5，0，-3|int||
|布尔值|true，false|bool||
|浮点值|1.2，3.4|double（ _not float！_ ）||
|字符|'a'，'\n'|char||
|C 风格字符串|"hello,world"|const char[]|默认添加 \0|

2. 改变推导出的字面量type，使用 字面量后缀 

|类型|后缀|含义|
|---|---|---|
|整型|u|unsigned int|
|整型|l|long|
|整型|ul|unsigned long|
|整型|ll|long long|
|整型|ull|unsigned long long|
|浮点|f|float|
|浮点|l|long double|
|string|s|std::string|
|string|sv|std::string_view|

```cpp
// auto 借助 字面量后缀 推导
auto s0 { "haha" };  // 推导为 const char*
auto s1 { "goo"s };  // "goo"s 是 std::string 字面量, s1 被推导为 std::string
auto s2 { "moo"sv }; // "moo"sv 是 std::string_view 字面量, s2 被推导为 std::string_view
```

### 字符串字面量

**what**：
双引号包裹，以 `\0` 结尾的字符序列常量（本质是 `const char[]` C风格数组 / C风格字符串 ）
1. 相邻的字符串字面量会 自动拼接 
```cpp
"hello"                                     // 本质：const char * 或 const char[]
list<const char *> ls = {"hello", "world"}; // 这里只能用 const char *，不能用 char *
```

```cpp
const char created[] =
    "<html>"
    "<head><title>Created</title></head>"

    "</html>";
// 等价于
const char created[] = "<html><head><title>Created</title></head><body><h1>201 Created</h1></body></html>";f
```

## const

**what**：
cpp中的 类型限定符（作为类型一部分），指定所声明的对象 只读 属性（运行时不能修改值）
1. 编译期、运行期 求值（常量表达式 作为初始化值时，编译期求值）
2. 🔥 const 修饰规则 ：先左后右（示例见 const指针）

const修饰总结：
1. const变量
2. const指针
3. const引用
4. const对象
5. const成员变量
6. const成员函数

### 顶层const和底层const

**what**：
- 顶层const：修饰 自身
- 底层const：修饰 引用/指向的对象

> 指针 可以是 顶/底const
> 引用 可以是 底const

拷贝时，忽略 顶层const，保留 底层const 
```cpp
/* 顶层const：拷贝时忽略 */
const int a = 10;   // 顶层const
int b = a;          // 合法（忽略顶层const）

/* 底层const：拷贝时不忽略 */
const int* p = &a;  // 底层const（指针指向常量）
// int* q = p;      // FIXME 错误（不能忽略底层const）
const int* r = p;   // 合法（目标也有底层const）
```

## constexpr

**what**：
隐含const的特性（const的超集），要求 变量/函数的值 在编译期就能计算得到，用于 性能优化
  1. 编译期求值（常量表达式初始化时）
  2. 编译器提供 *编译期检查* 、以及 优化 （见：编译期优化）

分类
1. constexpr变量
2. constexpr函数

使用场景
1. 非类型模板实参 的初始值
2. std::array 和 C风格数组 的数组长度

### constexpr变量

**what**：
编译期常量，必须在编译期求值，使用 常量表达式 初始化 
  1. 包含 隐式的 const
  2. 内部链接性
  3. 默认不是 inline
```cpp
// 返回 非constexpr值 的函数是 非constexpr的
int five() {
    return 5;
}

int main() {
    constexpr double gravity { 9.8 }; // ok: 9.8 是常量表达式
    constexpr int sum { 4 + 5 };      // ok: 4 + 5 是常量表达式
    constexpr int something { sum };  // ok: sum 是常量表达式
    std::cout << "Enter your age: ";
    int age{};
    std::cin >> age;
    constexpr int myAge { age };      // FIXME age 不是常量表达式
    constexpr int f { five() };       // FIXME five() 的返回值不是常量表达式
 }
```

**why**：
```cpp
// 由于历史原因，用 常量表达式 初始化的 const int 是常量表达式
const int a{b};          // 不好判断a是否为常量表达式，得看b是不是常量表达式
const int c{func()};     // 不好判断c是否为常量表达式，得看func()是否为常量表达式
// NOTE 有没有让编译器来替我们判断的方法呢？有！使用 constexpr
constexpr int a{b};      // 若a不是常量表达式，则报错
```

### constexpr函数

**what**：
可以在编译期/运行期求值（取决于是否在 常量表达式 上下文）
1. 不包含 隐式的 const
2. 外部链接性
3. 默认 inline
```cpp
constexpr int greater(int x, int y) {
    return (x > y ? x : y);
}

int main() {
    std::cout << greater(5, 6) << '\n'; // greater(5, 6) 可能在编译时 或 运行时计算
    constexpr int g { greater(5, 6) };  // greater(5, 6) 此处为 常量表达式上下文，因此在编译时计算
    std::cout << g << '\n';             // prints 6
}
```
