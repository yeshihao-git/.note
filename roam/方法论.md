# 方法论
## 通用方法论

**主动回忆学习法**（检验记住了多少）
1. 看完一篇内容就复述下讲了啥

**费曼学习法** （检验理解深度）
1. 和豆包AI对话：以输出倒逼输入，“教小学生” 的场景，强迫自己用简单语言拆解复杂知识，使用 结构学习法 ：`what + why + how` 讲述知识
2. 用贴近生活的例子辅助讲解：检验对知识本质的理解
3. 核心循环：输出讲解→暴露知识盲点 / 模糊点→查阅资料修正认知→再次输出讲解，循环直至能让新手轻松听懂，最终实现从 “记住知识” 到 “掌握知识” 的转化

## 代码方法论

**调试学习**
1. 通过 debug 的方式理解代码的逻辑调用关系

**学习性测试**（学习第三方库 API）
1. 编写测试代码来理解第三方库的 API
2. 学习结束后，将第三方库 API 封装到自己的类中（可以使用 [[适配器模式#适配器模式|适配器模式]]）

> 相比直接使用 第三方库 API，将第三方库 API 封装为类后使用更易于统一的维护管理

**代码重写**
1. （代码量小的情况）通过 ==全部重写== 第三方代码库的方式来学习代码
2. （代码量大的情况）通过 ==局部重写== 第三方代码库的方式来学习代码
