# 构造函数

**what**：
特殊的 成员函数，用于 初始化类对象
1. 名称同类名且无返回类型
2. 成员初始化列表 用于成员初始化
	1) const 、 引用 成员只能在此初始化
    2) 初始化顺序：按类中定义的顺序初始化 
```cpp
class Foo {
private:
    int m_x{};
    int m_y{};

public:
    // FIXME 先初始化 m_x，但是 m_x 使用 m_y 初始化，因此出错
    Foo(int x, int y) : m_y { std::max(x, y) }, m_x { m_y } { }

    void print() const {
        std::cout << "Foo(" << m_x << ", " << m_y << ")\n";
    }
};

int main() {
    Foo foo { 6, 7 };
    foo.print();
}
```
3. 构造函数体 用于其他任务（eg：打开文件、数据库...）
4. 🔥 执行顺序 ：默认初始化器/默认初始化 -> 成员初始化列表 -> 构造函数体
```cpp
class Foo {
private:
    int m_x {};                           // 1. 默认初始化器 (后续被 6 覆盖)
    int m_y { 2 };                        // 1. 默认初始化器 (使用)
    int m_z;                              // 1. 默认初始化（随机值）

public:
    Foo(int x) : m_x { x } {              // 2. 成员初始化列表
        std::cout << "Foo constructed\n"; // 4. 构造函数体
    }

    void print() const {
        std::cout << "Foo(" << m_x << ", " << m_y << ", " << m_z << ")\n";
    }
};

int main() {
    Foo foo { 6 };
    foo.print();
}
```
5. 内存分配 ：编译器为对象 分配内存 -> 然后在为初始化的对象上 调用 构造函数 初始化

类别
1. 用户定义的构造函数
2. 默认构造函数
3. 委托构造函数
4. 转换构造函数
5. 拷贝构造函数
6. 移动构造函数
7. 初始化列表构造函数

## 默认构造函数

**what**：
用户未定义构造函数时，编译器会为我们合成 隐式默认构造函数（也能 =default 显式生成）
1. 所有参数都有默认值 的构造函数 是 默认构造函数，应避免歧义调用
```cpp
class Foo {
private:
    int m_x { };
    int m_y { };

public:
    Foo(int x=0, int y=0) : m_x { x } , m_y { y } {     // 这是默认构造函数
        std::cout << "Foo(" << m_x << ", " << m_y << ") constructed\n";
    }
};

int main() {
    Foo foo1{};     // 调用 Foo(int, int)
    Foo foo2{6, 7}; // 调用 Foo(int, int)
    Foo foo3;       // 调用 Foo(int, int)
}
``` 
2. 何时调用 默认构造函数？ 值初始化 和 默认初始化 时 
```cpp
Foo foo{}; // 值初始化, calls Foo() default constructor
Foo foo2;  // 默认初始化, calls Foo() default constructor
```
3. 用户定义的默认构造函数 和 （隐式默认构造函数 和 =default定义的默认构造函数） 初始化行为不一样
```cpp
类名() {}
类名(参数1=默认值1, 参数2=默认值2, ...) {}
```

## 委托构造函数
**what**：
（cpp11）构造函数的 成员初始化列表 中调用另一个构造函数，减少重复代码
```cpp
类名(参数1) : 类名{ 值1, 值2 } { }  // 委托 类名(参数1, 参数2)
类名(参数1, 参数2) : 成员1{ 参数1 }, 成员2{ 参数2 } { }
```

## 拷贝构造函数

**what**：
用相同类型的对象 初始化对象
  1. 📖 面试题 为什么必须用引用？ 避免无限递归
```cpp
class MyClass {
public:
    MyClass(MyClass obj) { /* ... */ } // 错误：按值传递
};

int main() {
    MyClass a;
    MyClass b(a); // FIXME 调用拷贝构造函数时，需要先拷贝 a 到 obj，又触发拷贝构造函数 无限递归
} 
```
  2. 无自定义拷贝构造函数，编译器合成 隐式拷贝构造函数 ：对 类成员 进行 浅拷贝 （也能用 =default ），需要 深拷贝 类成员，需要自定义拷贝构造函数
  3. 使用 =delete 禁止拷贝
```cpp
类名(const T&) {}   //  & 必须有，const可选但是建议
```

## 转换构造函数

**what**：
基本类型 -> 类 ，单参数（或 带默认值的多参数）的构造函数，单个参数类型 隐式转换 为类类型
1. 多参数（不带默认值）也可以发生 隐式转换（通过 列表初始化）
```cpp
class Foo {
public:
    // NOTE Foo(int x, int y = 1) 同理
    Foo(int x) {
        cout << "发生隐式转换：调用 Foo(int) -> " << x << '\n';
    }

    // NOTE Foo(int x = 1, int y = 1) 同理
    Foo(int x, int y ) {
        cout << "发生隐式转换：调用 Foo(int, int) -> " << x << ' ' << y << '\n';
    }
};

void printFoo(Foo f) {}

int main() {
    printFoo(5);       // 发生隐式转换：调用 Foo(int)
    printFoo({6, 7});  // 发生隐式转换：调用 Foo(int, int)
}
```
2. 隐式转换 只能发生一次
```cpp
class Employee {
public:
    Employee(std::string_view name) {}
};

void printEmployee(Employee e) {}

int main() {
    // const char* -> 隐式转换为 string_view -> 隐式转换为 Employee -> 调用 Employee(std::string_view)
    printEmployee("Joe"); // FIXME 需要2次隐式转换，但隐式转换只能发送1次
    // NOTE 解决方法：printEmployee("Jos"sv) 或 printEmployee(Employee{ "Joe" })
}
```
3. 避免非期望的隐式转换 ，使用 explicit（最佳实践）
```cpp
类型(参数1) {}
类型(参数1, 参数2=默认值...) {}
```

### explicit

**what**：
修饰构造函数，使其只能 *显式调用* ，防止意想不到的隐式转换
```cpp
class Foo {
public:
    explicit Foo() {}
    explicit Foo(int x) {}
};

Foo getFoo() {
    return Foo{ };   // ok
    return { };      // FIXME 不能隐式转换 initializer list 到 Foo()

    return Foo{ 5 }; // ok
    return 5;        // FIXME 不能隐式转换 int 到 Foo(int)
    return { 5 };    // FIXME 不能隐式转换 initializer list 到 Foo(int)
}

int main() {
    getFoo();
}
```

## 移动构造函数

**what**：
（cpp11）参数为右值引用的构造函数，通过 “窃取” 源右值对象的资源初始化新对象，避免拷贝开销
  1. 右值（或 std::move 将左值转右值）初始化对象时 调用
  2. 编译器合成 隐式移动构造 条件：
     1) 无自定义 拷贝构造、拷贝赋值、移动构造、移动赋值、析构函数
     2) 可移动的移动，其余复制
```cpp
类名(类名&&) {}
```

## 初始化列表构造函数

**what**：
（cpp11）参数为 `std::initializer_list<T>` 的构造函数，支持 列表初始化（优先级高于普通多参数构造函数）
```cpp
类名(std::initializer_list<T> list) {}

std::vector v{1,2,3}; // 示例
```
