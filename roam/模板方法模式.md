# 模板方法模式

**what**：
一种行为设计模式，基类定义固定的算法步骤，允许派生类重写其中的某些步骤

> **示例**：文件格式解析
> ![[Pasted image 20260618103152.png|324]]
> 程序需要支持多种文件格式解析（doc、pdf、csv），解析的步骤是固定的（打开 - 解析 - 关闭），打开和关闭这两个步骤是相同的，只有 解析 这一步是不同的

**how**：
1. 基类定义算法流程（不变），其中的部分步骤需要交给派生类重写（可变）

> **示例**：
> 基类存在方法 A、B、C、D，方法 A 调用 B、C、D 构成算法流程，其中步骤 B 需要派生类重写实现特定功能，因此 B 设置为虚函数；
> 方法 A 为 public，方法 B、C、D 为 protected

![[Pasted image 20260618103119.png|592]]

```cpp
#include <string>
#include <iostream>
#include <memory>
#include <vector>

class FileParse
{
public:
	virtual ~FileParse() = default;
	
    void templateMethod() const
    {
        open();
        parse();
        close();
    }

protected:
    void open() const
    {
        std::cout << "open file" << std::endl;
    }

    virtual void parse() const = 0;

    void close() const
    {
        std::cout << "close file" << std::endl;
    }
};

class PDFFileParse : public FileParse
{
protected:
    void parse() const override
    {
        std::cout << "parse PDF file" << std::endl;
    }
};

class DOCFileParse : public FileParse
{
protected:
    void parse() const override
    {
        std::cout << "parse DOC file" << std::endl;
    }
};

class CSVFileParse : public FileParse
{
protected:
    void parse() const override
    {
        std::cout << "parse CSV file" << std::endl;
    }
};

void clientCode(std::unique_ptr<FileParse> fileParse)
{
    fileParse->templateMethod();
}

int main()
{
    auto pdfFileParse = std::make_unique<PDFFileParse>();
    clientCode(std::move(pdfFileParse));
    std::cout << "===============================" << std::endl;

    auto docFileParse = std::make_unique<DOCFileParse>();
    clientCode(std::move(docFileParse));
    std::cout << "===============================" << std::endl;

    auto csvFileParse = std::make_unique<CSVFileParse>();
    clientCode(std::move(csvFileParse));
}
```
