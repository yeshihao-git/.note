# 简单工厂

**what：**
工厂类中有静态工厂方法，根据传入枚举类型创建对应产品
1. 需要修改工厂类来添加新产品，不符合开闭原则
2. 适合 产品少且无需扩展 的情况

**how**：
1. 抽象产品派生具体产品
2. 工厂类中包含 产品枚举类型、静态创建方法（根据 产品枚举类型+switch 创建产品）

```cpp
#include <iostream>
#include <string>
#include <memory>
#include <stdexcept>

class Product
{
public:
    virtual ~Product() = default;
    virtual std::string someOperation() = 0;
};

class ConcreteProductA : public Product
{
public:
    std::string someOperation() override
    {
        return "concreteProductA";
    }
};

class ConcreteProductB : public Product
{
public:
    std::string someOperation() override
    {
        return "concreteProductB";
    }
};

class Factory
{
public:
    enum class ProductType
    {
        ConcreteProductA,
        ConcreteProductB
    };

    static std::unique_ptr<Product> createProduct(Factory::ProductType type)
    {
        switch (type)
        {
        case ProductType::ConcreteProductA:
            return std::make_unique<ConcreteProductA>();
        case ProductType::ConcreteProductB:
            return std::make_unique<ConcreteProductB>();
        default:
            throw std::invalid_argument("不支持的产品类型");
        }
    }

private:
    Factory() = default;
};

void clientCode(Factory::ProductType type)
{
    try
    {
        std::unique_ptr<Product> product = Factory::createProduct(type);
        std::cout << product->someOperation() << std::endl;
    }
    catch (const std::exception &e)
    {
        std::cerr << "创建产品失败：" << e.what() << '\n';
    }
}

int main(int, char **)
{
    clientCode(Factory::ProductType::ConcreteProductA);
    clientCode(Factory::ProductType::ConcreteProductB);
}
```
