# 类型别名

**what**：
为现有类型创建新名称，两种方式： using 和 typedef
```cpp
using 别名 = 原类型;  // 现代 C++ 风格（推荐）
typedef 原类型 别名;  // 古代 C 风格

using FcnType = int(*)(double, char); // 可读性更好，FcnType 容易找到
typedef int (*FcnType)(double, char);
```

如何使用？[fn:1]
1. 平台无关编码，适配不同平台（eg：固定宽度整数类型）
```cpp
#ifdef INT_2_BYTES
  using int32_t = long;  // 2字节平台
#else
  using int32_t = int;   // 4字节平台
#endif
```
2. 简化复杂类型
```cpp
using VectPairSI = std::vector<std::pair<std::string, int>>; // make VectPairSI an alias for this crazy type
bool hasDuplicates(VectPairSI pairlist) // use VectPairSI in a function parameter
{
  // some code here
  return false;
}
int main()
{
  VectPairSI pairlist; // instantiate a VectPairSI variable
  return 0;
}
```
3. 值的语义更清晰
```cpp
using TestScore = int;
TestScore gradeTest(); // 若这里返回 int，则不知道返回的是啥
```
4. 便于代码维护：一处修改，处处生效
```cpp
using StudentId = long;  // 只需修改此处的 long，所有使用 StudentId 的地方都会生效
```

## typedef

**what**：
古代 C 风格的类型别名方法
```cpp
typedef 原类型 别名;
```

## using

**what**：
（cpp11）现代 C++ 风格的类型别名方法
```cpp
using 别名 = 原类型;
```

最佳实践
1. 优先使用限定名称（eg：std::cout）
2. .cpp文件 中有限使用 using声明
3. 避免使用 using指示（除非测试代码中）

### using声明

**what**：
将命名空间中的 名称导入当前作用域 ，使用非限定名称访问
1. 导入后可参与 函数重载 [fn:2]
```cpp
using <命名空间>::<名称> // NOTE 语法

using std::cout;
cout << "Hello";
```

```cpp
class Base {
public:
    void print(int)    { std::cout << "Base::print(int)\n"; }
    void print(double) { std::cout << "Base::print(double)\n"; }
};

class Derived: public Base {
public:
    using Base::print; // 将基类的print加入派生类作用域，使其可以参与 函数重载解析
    void print(double) { std::cout << "Derived::print(double)"; }
};

int main() {
    Derived d{};
    // 参数是int，我们希望调用 Base::print(int)，若没有使用using，则会调用 Derived::print(double)
    d.print(5); // 调用Base::print(int)
}
```

**what**：限定名称
使用 作用域解析运算符（::） 将名称和命名空间关联的名称

**what**：非限定名称
名称和命名空间没有关联的名称
```cpp
std::cout // 限定名称
cout      // 非限定名称
```

### using指示

**what**：
`using namespace <命名空间>`将命名空间中的名称全部导入当前作用域，通过非限定名称访问全部内容：
```cpp
using namespace std; // 之后可直接用 cout、cin 等
cout << "Hello";
```

# 固定宽度整数类型

**what**：
（cpp11）能保证在任何平台都具有精确比特数的整数类型，解决 int/long 等类型宽度随平台变化的问题（cpp标准只限定int最小大小，没限定实际大小）。 本质 上是现有类型的 类型别名
```cpp
std::int#_t     // 有符号
std::uint#_t    // 无符号
// # 为位数（8、16、32、64）
```

如何使用？
1. int范围不重要：int（eg：年龄）
2. int范围重要  ：固定宽度类型

# std::size_t

**what**：
用于表示 系统最大对象大小 / 数组索引 的 无符号整数类型别名 （足够容纳当前系统任意对象的字节大小），通常是 `unsigned long` 或 `unsigned long long`
